Technical SEO issues
CriticalNo structured data / schema markup anywhere on the site
Person, LocalBusiness, Service, FAQPage, BreadcrumbList — all missing. Google cannot generate rich results (star ratings, FAQ dropdowns, sitelinks) without schema. Competitors with schema get higher CTR even at same ranking.
Fix: Add JSON-LD on homepage — Person + LocalBusiness (serviceArea: Noida, Delhi NCR, Dubai). Add FAQPage schema to each service page. Add BreadcrumbList to inner pages. Implement in 1 day in Laravel blade templates.
CriticalOG image is the personal photo (aruntyagi1.jpg) — not a designed card
og:image = personal photo used for all pages. When any page is shared on WhatsApp / LinkedIn / Twitter, the same portrait appears regardless of content. No branding, no keyword, poor first impression.
Fix: Create a 1200×630 branded OG image template per page type (homepage / service / blog). Laravel can auto-generate dynamic OG images using Browsershot or a service like og-image.vercel.app.
CriticalCore Web Vitals status unverified — hero image likely LCP offender
Homepage hero loads aruntyagi1.jpg. Without WebP + preload + fetchpriority="high", LCP is likely 3–5s on mobile (Indian 4G). Google uses mobile-first indexing. LCP > 2.5s = ranking penalty.
Fix: Run PageSpeed Insights on mobile now. Convert hero to WebP. Add <link rel="preload"> in <head>. Add fetchpriority="high" to the img tag. Target LCP < 2.5s.
MediumTitle tag and H1 are mismatched across the site
Homepage title: "Freelance Laravel Developer in India, Noida & Dubai" — H1: "Freelance Laravel Developer in Noida, Delhi NCR & India". Different keyword order confuses Google's primary topic signal. Same issue likely exists on inner pages.
Fix: Align title + H1 on every page. Lead with the single most important keyword. Homepage → "Freelance Laravel Developer in Noida, India | Arun Tyagi".
MediumMeta keywords tag still present (obsolete)
meta name="keywords" is present with 5 values. Google has ignored this since 2009. Not harmful but is a signal that the SEO setup was copy-pasted from old templates.
Fix: Remove meta keywords tag. Redirect that effort to meta description CTR optimisation.
Mediumrobots.txt and sitemap.xml not confirmed — cannot verify via fetch
Could not directly confirm sitemap.xml is properly submitted or that robots.txt references it. Need Google Search Console verification.
Fix: Open Google Search Console → Sitemaps → submit aruntyagi.com/sitemap.xml. Verify robots.txt has "Sitemap: https://aruntyagi.com/sitemap.xml" line. Check for any accidental Disallow rules.
MediumCSRF token exposed in HTML meta tag
meta name="csrf-token" is rendered in the page <head> and visible in source. While this is common in Laravel apps, it's unnecessary for a mostly-static marketing site and signals to crawlers that this is a dynamic app page.
Fix: Only inject CSRF meta tag on pages with forms. Remove from SEO landing pages where it's not needed.
LowCanonical URL on homepage is correct
canonical: https://aruntyagi.com/ — correctly set. Verify all inner pages also have canonical self-referencing tags to prevent duplicate content issues.
Fix: Audit 5 random inner pages — confirm canonical is set and matches the page URL exactly (no trailing slash inconsistencies).

On-page SEO issues
CriticalHomepage targets too many locations simultaneously
Noida + Delhi NCR + India + Dubai + UAE + Singapore + US are all targeted on one page. Google cannot rank one page strongly for 7 different geo-intents. Signal dilution is the primary reason homepage doesn't appear in top 10 for any single location keyword.
Fix: Homepage → target ONE primary intent: "freelance laravel developer in India" (broadest, highest volume). All other locations → dedicated inner pages (which already partially exist — expand them).
CriticalHomepage H2 hierarchy is scattered — 8+ competing topics
H2s on homepage: "Key statistics", "About Me", "Laravel & Full-Stack Development Services", "Hire by role", "Recent Projects", "How I Work", "What Clients Say", "Common Questions", "Let's Build Something". Each H2 pulls Google's attention in a different direction. No topical focus.
Fix: Homepage should have 2–3 H2s max, all supporting the primary keyword. Move "About Me", "How I Work", "Key statistics" to /about. Keep: "Laravel Development Services", "Recent Projects", "What Clients Say". The homepage is a conversion page, not a Wikipedia article.
CriticalService pages have thin content and weak keyword targeting
Pages like /php-development, /laravel-development appear in index but their meta descriptions reference Dubai/UAE on what should be India-focused pages. Content appears auto-generated or template-duplicated across service pages.
Fix: Each service page needs: 800+ words of unique content, clear H1 with geo + service keyword, 3–5 FAQs with FAQPage schema, 1 real case study or portfolio link, and a primary CTA. Prioritise /laravel-development and /web-development first.
MediumAlt text on hero image is too long and keyword-stuffed
alt="Arun Tyagi — freelance Laravel developer in Noida, Delhi NCR and India" — this is 9 words. Google's image alt should be descriptive (max 5–6 words) and not duplicate the H1 verbatim.
Fix: alt="Arun Tyagi — freelance Laravel developer, Noida" — concise and accurate. Then use the freed alt budget on portfolio/project images which likely have generic alts.
MediumInternal linking from homepage to money pages is weak
Homepage body text links primarily through the nav menu. Contextual in-body links (within paragraph text) to /laravel-developer-noida, /laravel-developer-dubai, /saas-development are missing or buried. Nav links pass less PageRank than contextual links.
Fix: Add 3–4 contextual links in homepage body copy: e.g. "If you're based in Noida or Delhi NCR, [see my local Laravel services →]". These should use exact keyword anchor text.
MediumBlog content is indexed but not internally linked from service pages
Blog posts (React developer Bengaluru, GA4 setup, etc.) exist but service pages don't link back to related blogs. This breaks the topic cluster model — Google needs to see the connection between service pages and supporting blog content.
Fix: Add "Related reading" section at bottom of each service page linking to 2–3 relevant blog posts. Example: /laravel-development → links to "Advanced Laravel techniques", "SaaS backend best practices".
LowFAQ section on homepage is good — needs schema
11 FAQ items are well-written and address real buyer questions. However without FAQPage schema, Google won't show them as rich results (the expandable FAQ snippet in SERPs). This costs significant SERP real estate.
Fix: Add FAQPage JSON-LD schema matching all 11 FAQ items. This is 20 min of work for potentially 3× SERP click area.

UX issues affecting SEO & conversion
CriticalHomepage is too long — primary CTA buried below the fold
The homepage has: stats → about → services → hire by role → portfolio → clients → process → testimonials → FAQ → contact form. The "Book a free call" CTA appears in the hero, but then the user scrolls through 10+ sections before reaching the contact form at the bottom. Bounce rate is likely high because it takes too long to reach the point.
Fix: Move contact form or "Book a call" CTA to immediately after the portfolio section. Cut the homepage to 5 sections: Hero → Services (3 lines each) → Portfolio (3 projects) → Testimonials (3) → CTA. Remove "Key statistics", "About Me", "How I Work" — these belong on /about.
CriticalNavigation has 13+ links — cognitive overload
The nav expands to 13 service links (Laravel Development, PHP Development, Laravel Developer, Web Developer, React JS Development, React JS Developer, React Native, Website Developer, Next.js, Mobile App, PHP Developer, Node.js) — presented as a flat list. Users cannot process this. High bounce probability on mobile.
Fix: Simplify nav to: Home | Services (dropdown: 4 top services only) | Portfolio | Blog | Book a call (CTA button, highlighted). Remove low-priority service pages from primary nav — they still exist for SEO, just not in nav.
Medium"Currently accepting 2 new clients for Q3 2026" — scarcity signal is good but not prominent enough
This is one of the most powerful conversion signals on the site and it's in a small top bar that most users scroll past. It communicates quality and creates urgency — needs to be larger and inside the hero section.
Fix: Add this as a bold line inside the hero — above the H1. "Currently accepting 2 new clients for Q3 2026 — [Book your slot →]". This framing has been shown to increase conversion rates 15–25% on freelancer sites.
MediumContact form has 6 dropdown fields before the message — too much friction
The form asks: Name, Email, Phone, Project type (dropdown), Budget range (dropdown), Timeline (dropdown), How did you find me (dropdown), Subject, Message — 9 fields. Most leads will abandon after the 3rd field. The dropdowns add cognitive friction without qualifying leads better than a 2-field form.
Fix: Primary form: Name + Email + Message only. Let Calendly / book-a-call handle the qualification. Or move the dropdowns to a 2nd step (after submit). This alone can 2× your form completion rate.
MediumTestimonials are not linked to verifiable sources
5 testimonials displayed — "Prerna, CEO, IALM Academy", "Ahmed Al-Rashid, Director, Dubai Charity" etc. — but no LinkedIn profile links, no Google review links, no profile photos. In 2026, unverified testimonials are heavily discounted by visitors (especially B2B buyers).
Fix: Add LinkedIn profile links to each testimonial. Add a real profile photo. Link to the live project. Add a "View on LinkedIn" icon. This transforms testimonials from "claims" to "social proof".
LowSkills section uses percentage bars with no context
"Laravel 90%", "PHP 95%", "Docker 65%" — these skill bars convey no real information. Who set these numbers? What does 90% Laravel mean vs 95% PHP? Visitors don't trust self-reported bars.
Fix: Replace skill bars with project-based proof. "Built 40+ Laravel applications including fintech, SaaS, and charity platforms." Numbers with context beat abstract percentages.
